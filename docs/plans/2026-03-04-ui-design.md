# HRMS 人事管理系统 UI 设计方案 (Phase 1)
**Date**: 2026-03-04
**Target Modules**: D 登录页、C 组织架构

## 1. 整体架构与风格设定
秉承系统设计规范（类飞书/钉钉现代化管理端布局），本次 UI 设计敲定了后续开发的核心视觉基调：
*   **主题色彩**：`#4F46E5`（科技蓝紫）为主色调。
*   **视觉流派**：极简响应式、卡片化布局，去线留白。
*   **交互原则**：信息层级清晰、深色导航辅助沉浸感、数据图表/列表文字绝对居中对齐。

---

## 2. D 登录页 (Login Page) 设计规范

### 2.1 布局结构：经典左右分栏模式
为了兼顾 B 端系统的稳重与现代感，排除了过于跳脱的居中悬浮和毛玻璃效果，最终确立了**左右分栏布局**：

*   **左侧品牌展示区**：
    *   **背景**：采用高质量的 3D 几何图形抽象背景（现代企业质感，包含浅蓝与紫色的柔和光影）。
    *   **文案表达**：左侧底部对齐展示系统 Logo `人事管理系统 (HRMS)` 及核心使命 `驱动组织效能 · 赋能企业创新`。
*   **右侧表单填充区**：
    *   **基础设定**：纯白色背景，无边框和阴影，提供最干净的阅读输入环境。
    *   **功能布局**：
        1.  顶部大标题 `系统登录` 与副标题引导。
        2.  账号、密码输入框（12px 圆角，浅灰色描边）。
        3.  提供 `记住密码` 与 `忘记密码？` 的实用项布局。
        4.  主按钮 `登 录` 填充品牌主色 `#4F46E5`。

---

## 3. C 组织架构 (Organization Structure) 设计规范

### 3.1 布局结构：左导航 + 左树右表
处理复杂的层级关系和人员调动，高度聚焦管理效率。

*   **深色左侧全局导航栏 (Sidebar)**：
    *   **背景色**：`#0f172a`（深灰蓝），提升专业感。
    *   **菜单项**：统一的 Icon + 文本。当前选中项（“组织架构”）使用 `#4F46E5` 高亮底色。
*   **右侧顶栏 (Header)**：
    *   包含当前页面路径面包屑（`首页 / 组织架构`）及极简的用户头像模块。
*   **内容核心区**：
    *   **左侧：部门层级树 (Tree Card)**
        *   纯白底色带极细描边边框。
        *   清晰展示 `总公司 -> 一级部门 -> 二级部门` 关系。选中状态的部门以浅紫底色及主色文字高亮。
    *   **右侧：数据与人员名单 (Data Card)**
        *   **头部**：居中展示当前查看的部门名称与人数总计，右侧绝对定位放置 `新增子部门` 与 `+ 添加成员` 的核心操作按钮。
        *   **表格区**：严谨的数据表格。表头及所有单元格文本（姓名、工号、职位、状态、操作）**强制绝对居中对齐**，确保数据对齐美感。状态字段（如“在职”）使用浅绿色胶囊样式进行视觉强化。

## 4. 后续开发建议 (Transition to Implementation)
1.  **CSS 框架**：当前视图中大量使用了类似 `padding: 24`, `fill: #f8fafc`, `gap: 16` 的设定，这与 **Tailwind CSS** 的 Utility-first 原则高度契合。前端开发时建议优先通过 Tailwind 映射这些空间与色彩令牌。
2.  **Vue 组件库**：Element Plus 的 `<el-tree>`, `<el-table>`, `<el-menu>` 可以直接承载上述设计，仅需覆盖部分默认样式（如表格文字统一 `align="center"`、树节点选中底色）。

后端分层架构（每个业务模块内部）
hr-xxx/
├── controller/       # 接口层（REST API，参数校验）
├── service/          # 业务逻辑层（接口 + 实现）
│   └── impl/
├── mapper/           # 数据访问层（MyBatis-Plus Mapper）
├── domain/
│   ├── entity/       # 数据库实体类
│   ├── dto/          # 数据传输对象（请求/响应）
│   ├── vo/           # 视图对象（前端展示）
│   └── query/        # 查询条件对象
├── convert/          # MapStruct 对象转换器
└── config/           # 模块级配置类
统一规范
统一响应格式：Result<T> 包装（code / message / data）
统一异常处理：GlobalExceptionHandler 全局异常捕获
统一分页：MyBatis-Plus 分页插件 + PageQuery / PageResult 封装
逻辑删除：所有业务表使用 del_flag 字段逻辑删除
审计字段：所有表包含 create_by / create_time / update_by / update_time
数据权限：通过自定义注解 @DataScope 实现部门级数据隔离

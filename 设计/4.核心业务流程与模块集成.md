📐 设计部分 4/5：核心业务流程与模块集成
1. 认证与权限流程
登录流程：
用户输入账号密码 → 后端验证 → 生成 JWT（AccessToken 2h + RefreshToken 7d）
→ 返回 Token + 用户信息 + 菜单权限 → 前端动态生成路由
请求鉴权：
前端请求 → Axios 拦截器附加 Token → 后端 JwtAuthenticationFilter 验证
→ AccessToken 过期 → 前端自动用 RefreshToken 刷新 → 无感续期
权限控制：
- 接口权限：`@PreAuthorize("hasPermission('employee:list')")` 注解
- 按钮权限：前端 `v-permission="['employee:add']"` 指令
- 数据权限：`@DataScope(deptAlias = "d")` 注解，SQL 自动拼接部门过滤条件

3. Flowable 工作流集成方案
统一的审批服务层设计：

java
// 统一工作流服务接口
public interface WorkflowService {
    // 发起流程
    String startProcess(String processKey, String businessId, Map<String, Object> variables);
    // 审批通过
    void approve(String taskId, String comment);
    // 审批驳回
    void reject(String taskId, String comment);
    // 转办
    void delegate(String taskId, String targetUserId);
    // 撤回
    void revoke(String processInstanceId);
    // 查询待办
    PageResult<TaskVO> queryTodoTasks(TaskQuery query);
    // 查询已办
    PageResult<TaskVO> queryDoneTasks(TaskQuery query);
    // 查询流程历史
    List<FlowRecordVO> getFlowHistory(String processInstanceId);
}
接入审批的业务场景：

业务场景	流程 Key	典型审批链
请假申请	leave_apply	员工 → 部门主管 → HR 专员
加班申请	overtime_apply	员工 → 部门主管
补卡申请	recheck_apply	员工 → 部门主管
招聘需求	recruit_demand	部门经理 → HR 经理 → 总经理（≥5人）
录用审批	offer_approve	HR 专员 → HR 经理
调岗申请	transfer_apply	员工 → 原部门主管 → 新部门主管 → HR
调薪申请	salary_adjust	部门经理 → HR 经理 → 总经理
晋升申请	promotion_apply	部门经理 → HR 经理 → 总经理
离职申请	resign_apply	员工 → 部门主管 → HR 专员 → HR 经理

模块间调用方式：由于是单体应用，模块间通过 Spring Bean 注入 直接调用 Service 接口，无需 RPC/HTTP。为保持解耦，各模块对外暴露 Facade 接口：

java
// hr-employee 对外暴露的 Facade
public interface EmployeeFacade {
    EmployeeSimpleDTO getEmployee(Long empId);
    List<EmployeeSimpleDTO> listByDeptId(Long deptId);
    DeptSimpleDTO getDept(Long deptId);
}
5. 文件存储方案
文件上传流程：
前端上传 → 后端接收 → 存入 MinIO（按 bucket 分类）→ 返回文件 URL
Bucket 规划：
├── avatars/        # 员工头像
├── resumes/        # 简历附件
├── contracts/      # 合同文件
├── certificates/   # 证件照片
├── training/       # 培训资料
└── exports/        # 导出文件（临时）

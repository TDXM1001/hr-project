📐 设计部分 5/5：非功能性需求、部署与测试策略
1. 安全设计
安全措施	实现方式
密码加密	BCrypt 单向加密存储
接口防重	自定义 @RepeatSubmit 注解 + Redis 分布式锁
XSS 防护	请求参数过滤（HtmlUtils 转义）
SQL 注入	MyBatis-Plus 参数绑定（不拼接 SQL）
敏感数据	身份证号、手机号脱敏显示（@Desensitize 注解）
操作审计	关键操作记录日志（@Log 注解 + AOP）
登录安全	验证码（图形 / 滑块）+ 登录失败锁定（5 次错误锁 30 分钟）
Token 安全	AccessToken 短时效 + RefreshToken 长时效，支持强制下线
文件安全	上传文件类型白名单、大小限制、文件名防注入
2. 性能设计
优化策略	实现方式
数据字典缓存	Redis 缓存字典数据，修改后自动刷新
用户权限缓存	Redis 缓存用户菜单/权限，登录时加载
分页查询	MyBatis-Plus 分页插件，避免全表扫描
慢 SQL 监控	MyBatis-Plus 性能分析插件 + Druid 数据源监控
大数据量导出	EasyExcel 流式写入，避免 OOM
薪资批量计算	异步任务 + 进度查询（@Async + SSE 推送进度）
数据库索引	高频查询字段建立索引，定期分析慢查询优化
接口限流	Guava RateLimiter 或 Redis + Lua 限流
3. 环境规划
环境	用途	配置
dev	本地开发	H2 内存数据库 / 本地 MySQL，嵌入式 Redis
test	测试环境	独立 MySQL + Redis，自动化测试
prod	生产环境	MySQL（主从）+ Redis（哨兵）+ MinIO + Nginx

5. 测试策略

6. 开发规范
规范	说明
Git 分支	main（生产）+ develop（开发）+ feature/*（功能）+ hotfix/*（修复）
提交规范	feat: 添加xxx / fix: 修复xxx / refactor: 重构xxx
代码风格	后端遵循阿里巴巴 Java 开发规范，前端遵循 ESLint + Prettier
API 规范	RESTful 风格，统一前缀 /api/v1/{module}/...
接口版本	URL 版本号（/api/v1/），便于未来升级
